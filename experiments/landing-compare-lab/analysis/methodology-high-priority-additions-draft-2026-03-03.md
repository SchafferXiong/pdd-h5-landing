## 1) 输入/输出标准化（最小素材包 + 标准产物）
建议插入位置：行动依据文档 `3.6 可配置框架` 前后。

新增条目草案：
- 输入（最小素材包）：
  `AppName/Icon/平台/下载方式`、`3-5条核心利益点`、`可信背书`、`1-3张素材或KV`、`合规限制与禁用词`。
- 输出（标准产物）：
  `页面结构大纲`、`组件级内容JSON`、`Design Tokens`、`埋点方案`、`A/B变体策略`。

高优先原因：
- 能把“需求描述”转为“可直接生产”的统一接口，减少每次开工重对齐。

## 2) 首屏 4 秒硬约束：Brand + Value + CTA + Proof 同屏
建议插入位置：行动依据文档 `3.2 信息框架` 与 `3.3 结构框架`。

新增条目草案：
- 首屏4秒必须同时可见：
  `Brand`（品牌识别） + `Value`（结果导向主张） + `CTA`（主按钮） + `Proof`（可见背书）。
- 优先顺序：先结果，再解释；先降低不确定性，再推动点击。

高优先原因：
- 当前方法已有 Brand/Value/CTA，但 Proof 同屏可见是提升“立即信任”的关键缺口。

## 3) 心理说服链顺序模板（用于短版2-3屏）
建议插入位置：行动依据文档 `3.3 结构框架`。

新增条目草案：
- 推荐顺序：
  `Result` → `Proof` → `Mechanism(可选)` → `Ease` → `Steps` → `Risk/Compliance` → `Re-CTA`。
- 页面过短时可压缩为：
  `Result + Proof`（首屏）→ `Benefits/Steps`（二屏）→ `Risk + Re-CTA`（收口屏）。

高优先原因：
- 把“结构”升级成“可复用叙事路径”，对跨行业复刻最稳定。

## 4) 埋点与实验章节（从设计规范升级为增长系统）
建议插入位置：行动依据文档 `5. 验收标准` 之后新增章节。

新增条目草案：
- 核心埋点：
  `page_view`、`hero/sticky/bottom_cta_click`、`section_view`、`scroll_depth(25/50/75/90)`、`time_on_page`、`outlink_click`。
- 实验优先旋钮：
  `首屏价值主张角度`、`CTA文案与视觉权重`、`Proof类型与位置`。

高优先原因：
- 没有埋点与实验，方法论只能“做页面”，无法“持续提转化”。
