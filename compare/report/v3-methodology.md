# 拼多多 H5 V3 方法论沉淀（复刻阶段）

## 目标
在不接入视频的前提下，用图片、文字、小组件和动效实现“视频化感知”，并优先保证移动端可用性与转化链路稳定。

## 1. 视觉骨架法
- 基底：红金高对比背景。
- 中心：奖励舞台（金额卡）作为首屏视觉锚点。
- 环境：发光粒子与金币雨作为低频环境动效。
- 结果：用户首屏焦点稳定在“金额 + CTA”。

## 2. 信息节奏法
- 屏1：单主承诺（奖励金额 + 主利益点 + 主CTA）。
- 屏2：3-5 条利益点卡片，按节拍高亮。
- 屏3：倒计时 + 进度 + 实时领取流收口。
- 结果：短路径内完成“理解-信任-点击”。

## 3. 动效层级法
- 低频：背景光效与金币雨循环。
- 中频：利益点卡片和场景图节拍切换。
- 高频：CTA 强度等级脉冲（0/1/2）。
- 结果：注意力不丢失，CTA 始终在节奏中心。

## 4. 转化闭环法
- 顶部弱 CTA：降低首次点击门槛。
- 首屏强 CTA：4 秒节拍内至少一次冲刺态。
- 底部常驻 CTA：滚动过程持续可达。
- 结果：任意滚动位置都可直接触发下载。

## 5. 可信增强法
- 实时领取动态（姓名打码 + 金额）。
- 倒计时与领取进度条。
- 短句数字化文案（避免长段解释）。
- 结果：强化时效与社会证明。

## 6. 稳定布局法
- 舞台网格：内容按网格行排布，不依赖大面积绝对定位堆叠。
- 安全区：顶部/底部安全区与固定悬浮 CTA 协同。
- 高度合同：卡片最小高度、文字最多两行、按钮最小触控高度。
- 结果：360/390/430 三档下减少重叠、遮挡、越界。

## 落地到 V3 Replica 的组件映射
- 屏1 Hero：`hero-stage` + `reward-panel` + `hero-copy` + `cta-main`
- 屏2 Benefits：`benefit-scene` + `benefit-list` + `cta-sub`
- 屏3 Urgency：`countdown` + `progress-track` + `live-feed` + `cta-main.compact`
- 全局转化：`mini-cta` + `sticky-cta` + 统一 `appendParams` 跳转函数
